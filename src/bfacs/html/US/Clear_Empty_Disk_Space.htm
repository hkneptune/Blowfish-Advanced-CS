Clear Empty Disk Space

With this dialog you can clear the whole empty file space on the one or multiple drives. The program will create a temporary file until the disk is full, overwriting every file information not in use anymore.

Note that clearing empty disk space makes file content recovery, even for good purposes, impossible!

The Setup allows you to tell Blowfish Advanced CS what and how to clear.

Write through Buffers

Creating temporary files with sizes of several GB will make every caching mechanism of Windows worthless. On the contrary caching might slow down the whole process or even lock your computer for a long time until all buffers have been written to disk. To avoid such problems activate the switch. This should work well for the most cases.

No Messages

After a drive has been cleared a message box will appear. The temporary file will be deleted after this box has been closed. Although Blowfish Advanced CS flushes all buffers before, it might help in some cases to give Windows (or even the drives' hardware) some extra time to write down the rest of the data. However, if you want to clear multiple drives such message boxes might get annoying and prevent you from doing the whole job unattended (e.g. during a coffee break). By turning this switch on you can get rid of the messages.

Low Priority

You can force Blowfish Advanced CS to lower its priority during the clearing process, enabling you to work with other applications (which usually run at a normal priority) without any significant delays. Please remember that you will get out of disk space at the end of the clearing, so watch out when trying to save files in other applications.

Clear

You can tell the software what to clear. Either just the current location, all drives (excluding CD-ROM, but including removable drives) or fixed drives (hard disks).

Clear with

The temporary file can either contain zero bytes or random data. For random data you can tell the software to create just one block and using it again and again (static random data) or putting out continuous random data, which is at last the slowest method, as the program must create data all the time, not only at the beginning. It’s is strongly recommended to use random data when dealing with compressed drives.

Click on the Start button to begin the clearing. If the size of empty disk space (on the current drive) can be determined a progress bar will appear on the right. Otherwise the number of bytes written is shown in the status bar. The process can always be terminated with the Cancel button.
