Key Cache

When working with encrypted documents most users don't change their keys after every action, so reentering the same key for encryption, decryption, viewing and other purposes is the most boring and time consuming part of the work.

Blowfish Advanced CS offers you a solution for this: if you have entered a key once you can let it be cached for a certain (or even an infinite time) in an internal buffer. To avoid the key being swapped out to the system's page file accidentally it is place in a special type of memory.

With a cached key you just have to confirm instead of entering it again. For some actions e.g. viewing encrypted files even confirmations are not necessary at all.

Having a secret key somewhere in the backyard should also handled a little bit with care. You should be the only one having access to the computer during this time. If you're leaving temporarily, e.g. for a coffee break just expire the cached key manually.
