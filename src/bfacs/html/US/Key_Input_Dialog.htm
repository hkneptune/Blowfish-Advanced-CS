Key Input

Here you put in your key consisting of a password or a key disk to start encryption or decryption actions on the files you selected.

The password can be everything, a single word or a complete sentence. The maximum length of a password is just limited by the edit control itself (which is usually around 32kB), assumed that you are running the registered version (with no key limitations) of Blowfish Advanced CS.

You can also type in the complete ASCII character set by using binary sequences. E.g. if you want to have the character with the ASCII code # 8 (that's the code for the backspace key) at the end of your password named "special" you just type "special\08". The "\08" sequence tells Blowfish Advanced CS to ignore the "\" prefix and to take the next two characters as the hexadecimal representation of the ASCII code. If you want to use the "\" alone you will have to type "\\". By this method the key spectrum can be extended easily and brute-force attacks will be much harder. You may even use zero bytes in your password, e.g. "sec\00ret".

If the password cannot be detected with the Auto Confirmation turned on or if Auto Confirmation is turned off you will have to enter the password (for encryption purposes only) a second time to avoid typing mistakes.

There will be a warning appearing in case the Caps Lock Key is enabled. This protects you from accidentally and unintentially entering passphrases with capital letters, one of the most common input errors made.

Please read the security aspects to avoid flaws when choosing your password.

Use the Apply button to fix changes permanently.

The following items are visible for every action:

Use Cached Key

If a key is available the key cache and you want to use it, so turn on this switch. All other key input controls will then be disabled.

Show Password

If your are sure that nobody (not even a hidden camera) has the possibility to look over your shoulder you can turn on this switch to see your password instead of the blank characters. Blank characters are much better to hide a password instead of the usual asterisks, because it is harder for someone to guess the length of the password when one has the chance to peep.

Auto Confirmation

Confirming a password again and again gets boring after a while. With Auto Confirmation turned on Blowfish Advanced CS will remember the password you entered and confirmed. If you enter the password later again the program detects it and continues without further requests. The key input dialog will be closed as soon as your password is recognized, so you mustn't even touch the enter key. This saves a lot of time and avoids typing errors which occur even with confirmations. After retyping a password again and again it gets so common that you may even duplicate a wrong key-stroke.

Cache Key

If you want the key you are entering to be put into the program's key cache then check this switch. Remember that when you are entering multiple keys the resulting big key will be put into the cache, not the single one.

Multi Key Input

Blowfish Advanced CS allows you to concatenate several keys. You can combine passwords and key file contents in any order you want to. This allows you e.g. to secure your key disk with an additional password (like a PIN number). If the key disk was stolen one will still need the password to have access to your data. Another example is to need two persons to access a secret document: both of them use their own password or key disk and are only able to decrypt the data together. Multi keys are entered by activating the switch. You will then be asked after every key input if another key is going to be entered. After all keys are provided they are linked together (in the order they were put in) and treated as one big single key. So if the first password e.g. was "123" and the second one "456" the resulting key is "123456". For decryption you may enter "123456" instead of two single keys, of course.

Use Key Disk

Activate this switch if you want to use a keydisk instead of a password. A keydisk is a (preferably removable) disk, usually a conventional 3.5" floppy disk, with a defined key file on it. By default this file is located at "A:\BFACSKEY", but you can change this setting in the options dialog. There you can also create a key disk with secure random data. Insert the key disk only at the time when Blowfish Advanced CS wants to read it. That will be shortly after you leave the key input dialog with the "OK" button. Using key files is the right choice for those people who have problems to remember or don't want to remember passwords. It is also a fine feature for sharing keys in a small workgroup. Please backup your key disks! If a key disk is lost your data encrypted with the key file will be gone. Keep key disks at a safe place! If someone has access to them one will be able to copy it and then decrypt all your encrypted data.

These items are only available for encryption:

Compress

Blowfish Advanced CS can compress file data before encrypting it. By setting this switch you can turn on data compression. If a file cannot be compressed, e.g. multimedia files like JPEG or archives like ZIP, it is stored in an non-compressed form to avoid wasting disk space. Please check this out, because the program tries to compress every file and starts a second time, with compression temporarily turned off. You may also tell the software what filetypes not to compress. Compression is time-consuming, due to that the whole process slows down. But on highly compressable documents, like text files or spreadsheet tables it does not only save disk space, but increases the security, too. Cryptanalysis on compressed data is much harder then on uncompressed data.

Store Original Pathnames

Storing pathnames is useful if you want to keep complete directory structures, but don't want to leave the files at their original locations. The original pathname of a file will be stored encrypted in the header of the cryptfile. If you decrypt the file its original path will be restored. E.g. if you encrypt the file "D:\Documents\Private\mydiary.doc" with pathname storing on and you decrypt the encrypted file (with activated pathname restoring) to the target path "E:\Backups" the original file will be recreated as "E:\Backups\Documents\Private\mydiary.doc".

Rename Files

Activating this option will cause a complete renaming of encrypted files, so nobody can predict anything about what was in your file when the original name and extension have vanished. Depending on the settings in the options dialog files will be renamed either with random characters or with an enumerated name masking. There you can also choose if the program should add the .BFA extension to encrypted files.

Relative To Current Path

This switch is only enabled if you store original pathnames. If it's checked pathnames will only be store relatively to the current path of the browser. E.g. if you're in "E:\Backups\Documents" and encrypt the file "E:\Backups\Documents\Private\mydiary.doc" the original path and name will be saved as "Private\mydiary.doc". Otherwise the complete path will be stored, in our case "Backups\Documents\Private\mydiary.doc".

For decryption only this item is available:

Restore Original Path

If you have encrypted files with their pathnames (either relatively to the base path or not) you can recreate the directory structures by checking this switch. If you let it unchecked, the original pathname will be ignored.

For encryption and decryption these items are available:

Keep Directories

If you are using a target path you can force the software to keep all directories and recreate them at the new location. E.g. if you are in "E:\Backups\Documents" and your target path is "D:\Documents" an encrypted file like "E:\Backups\Documents\Private\mydiary.doc.bfa" will be decrypted to "D:\Documents\Private\mydiary.doc". It is not recommended to combine this option with the restoring of pathnames for decryption.

Use Destination Path

By default files will be written at the same place where the original files (either encrypted or unencrypted) were located. If you want to have the files at another location, e.g. for backup purposes, you can type in a destination if the switch is turned on. Use the Browse... button to select a previously entered pathname from the list (or to browse for a new folder). In combination with the store and restore pathnames switches you can recreate complete directory structures where you want them to be.

Remove Source Files

If you encrypt or decrypt files to a destination the process is similar like copying with encryption or decryption afterwards. By activating this switch the source files will be wiped (for encryption) or deleted (for decryption purposes), so the process is similar to moving files.

For reencryption these items are available:

Old/New Encryption Algorithm

You can select a new encryption algorithm for the files or provide the previously chosen algorithm respectively. You may also use the same algorithm for reencryption, but then you should at last change the encryption key. Otherwise the results would be the same. Please remember that even if you're using the same key and don't change the algorithm the reencrypted files will be totally different in their content. This is because every time your key is combined with a random value (salt). For more information please read the technical reference.
